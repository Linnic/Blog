Hexo 操作日誌(一)

離架好網站拖了兩天，雖然內容早已打好，但要怎樣上傳，怎樣編寫內容還是個大問題。

Problems：

  1. git commit 一直沒有東西傳上去
  2. Ubuntu Server 只有 tty 的介面，無法切換中文語系，故不能直接顯示中文及在 .md 中進行中文修改
  3. 相對路徑會遇到”前面路徑被刪除”以及”後面路徑重複”，可能根據當前頁面而導致路徑出錯
  4. 編寫語言
  5. <!DOCTYPE html> 出現在文章中
  6. 在 Hexo server 開啟中傳送文章，並重整網頁會有遺失檔案的可能性

Solutions：

  1. 沒有下 hexo generate，所以 hexo 沒有產生靜態的檔案至 public，導致在上傳時沒有發現任何修改。
  2. 還是採用 PSFTP 將 ~/source/_posts/ 下的 .md 檔抓回，再用 Notepad or Sublime Text 進行修改，
    然後再傳回。
  3. 找到 generate 後，文章會被產生在 /public/2016/03/16/filename/ 下的 index.html 再重新設定相對路徑，但無解….。
    改採用 <!--more-->讓文章內容在首頁隱藏。
  4. 可用 html or Markdown，文章中的段落會被自動改以 <p></p> 包起來，’1. ‘ 會被改成 <ol><li></li></ol>。
  5. 由於會自動加入框架，所以 html 的大框架可以不用打。
  6. 避免此操作。
