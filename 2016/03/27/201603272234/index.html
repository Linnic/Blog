網路拓樸模擬(四) - 走出 Mininet

拖了快一個禮拜，本來這篇是最早打完的說…..
為了補前幾篇，還有圖片跟實驗操作硬是拖到最後……

下雨天~~ 金門依舊陰雨綿綿
今日 SDN 課堂教學，主要再講如何讓 Mininet 內的虛擬網路連至外部網路。

今天主要可分為：

1. 透過內部網路連結

a. 環境建置
b. 網路拓樸設定
c. 實驗操作

2. 透過 NAT 網路連結

a. 環境建置
b. 網路拓樸設定
c. 實驗操作

內部網路連結

環境建置

  1. 需要 2 台虛擬機，可將原本的 SDN_hub 用 VirtualBox 的再製，暫時再弄出一個可執行的虛擬機。

  2. SDN_Hub 需要 2 張網路卡，一張為預設 ( NAT )
    ，再新增一張，並將連線方式設為內部網路，名稱為 intnet_1。

  3. 再製後的虛擬機，將其網路連線方式設為內部網路，名稱一樣為 intnet_1。

網路拓樸設定

從 SDN_Hub 借 eth1 來給 router_1 使用，讓 Mininet 下的　host_1 可以透過 router_1 經過 intnet_1 可與再製的 SDN_Hub 做連結。
參考範例：example

實驗操作

  1. SDN_Hub

    # 查看 現在網路狀態，eth1 未被啟動
user:$ ifconfig

# 啟動 eth1
user:$ sudo ficonfig eth1 up

# 進入 mininet
user:$ chmod +x test_8.py
user:$ sudo ./test_8.py

# 查看 eth1 是否有正確被綁定，結果沒有 IP
mininet> ifconfig

# 重設 router_1 eth1 的 IP
mininet> router_1 sudo ip addr add 10.0.0.1/24 brd + dev eth1

  2. 再製的虛擬機

    user:$ sudo ip addr add 10.0.0.100/24 brd + dev eth0
user:$ sudo ip route add default via 10.0.0.1

  3. 檢查網路是否暢通

    # SDN_Hub
mininet> host_1 ping router_1
mininet> host_1 ping 10.0.0.100 -c 3

# 再製虛擬機
user_2:$ ping 192.168.0.10 -c 3

NAT 網路連結

環境建置

  1. SDN_Hub 只需要一張網卡，預設連線方式為 NAT。

網路拓樸設定

從 SDN_Hub 借 eth1 來給 router_1 使用，讓 Mininet 下的 host_1，可透過 router_1 再經由 NAT 網路連線到 Internet。
參考範例：example

實驗操作

  1. 進入 Terminal

    user:$ chmod +x test_9.py
user:$ sudo ./test_9.py

# 檢查設定
mininet> router_1 ifconfig

# 檢查網路是否暢通
mininet> router_1 ping  host_1
mininet> router_1 ping  8.8.8.8

mininet> host_1 ping router_1
mininet> host_1 ping 8.8.8.8

# 如果 router_1 可以 ping google 而 host_1 不行，就是 IP 沒轉成功，在下一次
mininet> router_1 iptables -t nat -A POSTROUTING -o eth0 -s 192.168.0.0/24 -j MASQUERADE

WARNING: 此程式結束後，SDN_Hub 的 eth0 就算用 sudo ifconfig eth0 up 也無法將 eth0 找回，懶人解法就是 reboot。

Reference：

Software Defined Network (SDN)—Mininet Learning Guide - 柯老師
