React-native 起手式 - 環境建置(Windows)

想想，也該來學習 React-native 了，安裝的過程多災多難~~~

本來在 Windows 上安裝不順利，cmd 沒有權限的問題，忘記可以按右鍵以系統管理員來開啟。

然後跟朋友抱怨了一下，就決定裝在 Virtualbox 下的 Ubuntu Server，好巧不巧，React-native 裝完後，在裝 Android studio 時，硬碟空間竟然不夠了~~~~一開始沒想那麼多，就想直接新增虛擬硬碟來擴充，然後發現最尷尬的……

GOD~~ 竟然跟另外一台 Server 共用虛擬打硬碟…….
腦中閃過~ “又要重灌了…..”

好吧，那就來灌，花了一天，把 nvm、node、Hexo、SSH、Git、Reacr-native、Android studio、VirtualBox、Genymotion、JDK-8，完整裝過一遍，當然，中間陸續有些問題，不過最後一次裝起來滿順手的。

一開始，React-native start 後就卡在這

[21:49:17] Building Dependency Graph (2909ms)

真的不知道該怎辦，後來發現發現應該是模擬器沒設定，然後把 Android 後面的套件都裝一裝。

本以為裝完人生從此順遂，可以開始 React-native 的道路時，才發現…… Genymotion 根本開不起模擬器 VirtualBox 也不行，我操…….

1. Genymotion 的問題:

a. DHCP & IP not set >> 試著去修改 VirtualBox，最後都會被 Genymotion 改回……
b. 還有 CPU 數量不夠 >> 調成 1 CPU 即可

2. VirtualBox 的問題:

a. this kernel requires an x86-64 CPU, but only detects an i686 CPU, unable to boot >> 需要 VT-X/AMD-V 功能開啟，偏偏我的電腦 BIOS 沒這功能….. VirtualBox 設定開啟也回天乏術 ……

各種內心小劇場想放棄啊 ~~~~ 擔心我畢業可能沒法面試上想要的公司……啊 ~ 啊 ~ 啊 ~ 啊 ~~

然後又抱怨了一下之後 …… 跑去睡覺 ~ 晚上睡醒沒吃飯，又恰逢下雨，就不出門，垃圾也不倒了，打開電腦試試看之前在 Windows 裝不起來的 React-native，想到用前面提到的方法來安裝，總算成功了 ~ 可喜可賀 ~

接著又花了好一段時間把 Android studio、JDK-8、Genymotion 都升級、重裝、環境變數設定，好不容易搞好，執行到 React-native start 又卡住了……。

爬了一下文，自己也整理了一下順序：

開啟步驟:

  1. 將專案匯入 Android studio

  2. 在 Genymotion 設定模擬器

  3. 啟動 React-Native Server

    react-native start

  4. 打開瀏覽器瀏覽，http://localhost:8081/index.android.bundle?platform=android
    看到 JS 文件代表成功，而 Building Dependency Graph 後沒下文的東西都會跑出來 ~ 灑花 ~~

  5. 用 Genymotion 啟動模擬器

  6. 打開另一個 cmd ，安裝 Debug

    cd AwesomeProject\android
gradlew.bat installDebug

  7. 後續等兩個 cmd 都跑到一定段落(Server 的 cmd 會一直開著不管，另一個跑完就沒事了……)

  8. 手動打開 App …… 感謝老天從來沒有拋棄我 ~~ 泣 ~~

好了，文章記錄到這 ~ 開始來救援之前的文章 ……
